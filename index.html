What is Docker?

Docker is a set of tools that allows you to package, distribute, and run applications inside a container. It is a platform for building, shipping, and running distributed applications that use Docker containers.

Docker helps you run your application anywhere ‚Äî without worrying about setup, versions, or dependencies.

‚ÄúIf it works on my computer, it works on yours too.‚Äù

Real Example:

Suppose your app needs:

Python 3.12

MySQL

Node.js 18

Some special libraries

Instead of forcing everyone to install all that, you can package everything (OS, code, dependencies) into one container.

What is Docker Compose?

Docker Compose is a tool for defining and running multi-container applications. It uses YAML files to configure services and their networks.

What is Docker Desktop?

Docker Desktop is a desktop application for running and managing Docker containers.

What is Docker Machine?

Docker Machine is a tool for creating and managing virtual machines with Docker.

What is Docker Swarm?

Docker Swarm is a tool for managing Docker containers in a distributed environment.

What is Docker Network?

Docker Network is a tool for creating and managing networks for Docker containers.

What is Docker Volume?

Docker Volume is a tool for creating and managing volumes for Docker containers.

What is Docker Image?

Docker Image is a tool for creating and managing images for Docker containers.

What is Docker Container?

Docker Container is a tool for creating and managing containers for Docker.

What is Dockerfile?

Dockerfile is a text file that contains instructions for building a Docker image.

What is Docker Hub?

Docker Hub is a public registry for Docker images. You can pull and push images from Docker Hub.

How Docker Works

Think of Docker like this:

Concept -------------- Explanation ------------- Analogy

Docker Image ------ Blueprint of your app (read-only) ------ Like a recipe

Docker Container ------ Running instance of that image ------ Like a cooked dish

Dockerfile ------ Instructions to build an image ------ Like the recipe card

Docker Hub ------ Cloud store of images ------ Like Play Store for Docker

Docker Engine ------ The runtime that runs containers ------ Like a chef who cooks

Step 2: Run Your First Container

Run the following command in your terminal to create a container called "hello-world":
docker run hello-world

Docker will:

1. Pull the "hello-world" image from Docker Hub. (Download the ‚Äúhello-world‚Äù image from Docker Hub)
2. Create a container based on the image.
3. Run the container.
4. Print the output of the container. (Run it and print ‚ÄúHello from Docker!‚Äù)

Step 5: Docker Compose (Multiple Containers)

If your app needs Node.js + MySQL, you can use Docker Compose to run both together.

Create a file: docker-compose.yml

In .yml file write:

version: '3.9'

services:

app:

build: .

ports:

- "3000:3000"

depends_on:

- db

db:

image: mysql:8

environment:

MYSQL_ROOT_PASSWORD: root

MYSQL_DATABASE: mydb



Run both with one command:

docker-compose up

To stop: docker-compose down
